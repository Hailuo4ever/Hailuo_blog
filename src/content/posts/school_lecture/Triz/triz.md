# TRIZ理论核心考点复习提纲

> 来源：根据 `ppt/` 目录下 14 份课程 PDF 课件梳理，包括 TRIZ 理论概述、专利知识、因果分析、功能分析、资源分析、最终理想解、技术矛盾、物理矛盾、物场分析、标准解、技术系统进化法则、科学效应知识库等内容。

## 0. 一条主线先背下来

TRIZ 课程的解题逻辑可以按 `A-D-S-V` 理解：

| 阶段       | 重点工具                                                     | 复习抓手                             |
| ---------- | ------------------------------------------------------------ | ------------------------------------ |
| 分析问题 A | 因果分析、系统功能分析、资源分析、S 曲线、多屏幕分析         | 找问题、找根因、找资源、找短板       |
| 定义问题 D | 最终理想解、技术矛盾、物理矛盾、物场模型、功能化模型         | 把原始问题转成 TRIZ 可处理的问题模型 |
| 解决问题 S | 40 个发明原理、4 大分离原理、76 个标准解、科学效应库、ARIZ、技术系统进化趋势、系统裁剪 | 按模型选工具，形成方案               |
| 评估方案 V | PUGH 矩阵、专利引用分析                                      | 对方案进行比较、筛选、验证           |

考试最可能考的不是“背概念”本身，而是：给一个工程现象，要求你识别问题类型、建立功能/矛盾/物场模型，再选择对应 TRIZ 工具。

## 1. TRIZ 概述

### 1.1 创新、发明、创造

- 创造：产生原始设想，可以是头脑影像、草图、模型等。
- 发明：原始设想经过技术可行性证明，例如计算、仿真、物理模型试验。
- 创新：发明完成商品化开发，并通过产品从市场获得收益。
- 课程中的公式化表达：`创新 = 设想 + 发明 + 商业开发`。

### 1.2 问题的定义和分类

- 问题：期望状态与当前状态之间存在的距离。
- 按来源分：
    - 发生型问题
    - 探索性问题
    - 假设型问题
- 按解决关键分：
    - 原因导向型问题
    - 目标导向型问题
- 按难易分：
    - 通常问题：关键步骤和知识已知，按传统经验可解。
    - 发明问题：至少有一个关键步骤未知，目标不清楚，或存在相互矛盾的需求。

### 1.3 TRIZ 的概念

- TRIZ 是“发明问题解决理论”，英文常写作 `Theory of Inventive Problems Solving`。
- 由苏联发明家根里奇·阿奇舒勒从 1946 年开始，在大量专利分析基础上总结形成。
- 本质：从大量专利和创新案例中抽象出解决发明问题的范式、原理和规律。

### 1.4 TRIZ 的核心思想

重点背三句：

1. 技术系统的发展遵循客观进化规律和模式。
2. 各种技术难题、冲突和矛盾的解决，是推动技术系统进化的动力。
3. 技术系统发展的理想状态，是用尽量少的资源实现尽量多的功能。

## 2. 专利知识

### 2.1 专利的基本概念

- 专利：向国家专利行政部门提出申请，经依法审查合格后，授予申请人在规定时间内对发明创造享有的专有权。
- 三类专利：
    - 发明：对产品、方法或其改进提出的新的技术方案。
    - 实用新型：对产品的形状、构造或其结合提出的适于实用的新的技术方案。只包括有形产品；方法、工艺、材料产品、软件本身不能申请实用新型。
    - 外观设计：对产品形状、图案、色彩或其结合所作出的富有美感并适于工业应用的新设计。

### 2.2 专利权的特点

| 特点   | 含义                                             |
| ------ | ------------------------------------------------ |
| 垄断性 | 有效期和法律管辖区域内，未经许可不得实施         |
| 时间性 | 发明保护 20 年；实用新型和外观设计保护 10 年     |
| 地域性 | 只在申请并授权的国家或地区有效                   |
| 公开性 | 技术资料必须公开到所属领域技术人员能够实现的程度 |

### 2.3 专利权主体

- 发明人或设计人：对发明创造的实质性特点作出创造性贡献的人。
- 申请人：提交专利申请的单位或个人。
- 专利权人：专利授权后的权利主体。
- 职务发明：执行本单位任务，或主要利用本单位物质技术条件完成的发明创造，申请专利的权利通常属于单位。

### 2.4 专利申请文件

常考清单：

- 请求书
- 说明书
- 权利要求书
- 说明书附图
- 说明书摘要、摘要附图
- 外观设计图片或照片
- 外观设计简要说明

说明书结构重点：

- 名称：简短，通常不超过 25 字，不能含商业宣传语。
- 技术领域：说明所属技术领域。
- 背景技术：说明已有技术及其问题。
- 发明内容：写要解决的技术问题、技术方案、有益效果。
- 附图说明：说明各幅附图。
- 具体实施方式：说明优选实现方式。

权利要求书的作用：

- 用技术特征总和表示要求保护的技术方案。
- 判断专利性。
- 确定专利权保护范围。
- 判断侵权或被侵权。

### 2.5 授权条件

| 条件   | 核心含义                                                     | 易考点                                                       |
| ------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 新颖性 | 不属于现有技术，也没有抵触申请                               | 现有技术包括申请日前国内外公众所知技术；新颖性审查采用单独对比 |
| 创造性 | 发明具有突出的实质性特点和显著进步；实用新型具有实质性特点和进步 | 常用“三步法”判断                                             |
| 实用性 | 能制造或使用，并产生积极效果                                 | 永动机、不可重复、无积极效果等不具备实用性                   |

申请文件还要满足：

- 单一性
- 权利要求书得到说明书支持
- 权利要求清楚简要
- 说明书充分公开
- 修改不能超出原申请文件公开范围

### 2.6 审查意见答复

对不具有创造性的审查意见，答复三步法：

1. 确定最接近的现有技术。
2. 确定发明的区别技术特征和实际解决的技术问题。
3. 判断区别技术特征对本领域技术人员是否显而易见。

### 2.7 专利挖掘

专利挖掘本质：找出能够解决一定技术问题、带来一定技术效果、且与现有技术有差别的技术方案。

常用操作：

- 从研发任务出发：任务拆分为子任务，再分析技术要素和创新点。
- 从特定创新点出发：横向、纵向扩展关联因素，形成新的技术方案。
- 注意保密、避免在先公开、处理抵触申请问题。

## 3. 因果分析

### 3.1 5WHY 分析法

定义：从结果出发，沿因果链连续追问“为什么”，直到找到根本原因。虽然叫 5WHY，不限于只问 5 次。

开始前准备：

- 整理和细分问题，避免把多个独立问题混在一起。
- 区分问题发现场所和问题发生场所。
- 用 `5W1H` 掌握现状：What、Where、When、Who、Why、How。
- 关注 `5M1E`：人、机、料、法、环、测。
- 理解相关系统的构造和功能。

步骤：

1. 说明问题并描述相关信息。
2. 连续追问为什么，直到找出根本原因。
3. 制定并执行对策。
4. 执行后验证有效性。

### 3.2 因果链分析法

定义：从初始缺点出发，逐层分析造成问题的直接原因，构建因果链条，是 5WHY 的延伸和扩展。

关键术语：

| 术语     | 含义                                                         |
| -------- | ------------------------------------------------------------ |
| 初始缺点 | 由项目目标决定，通常是项目目标的反面                         |
| 中间缺点 | 处于初始缺点和末端缺点之间，既是上层原因又是下层结果         |
| 末端缺点 | 因果链分析停止处，例如物理极限、自然现象、法规限制、无法继续找到原因等 |
| 关键缺点 | 从因果链中筛选出的、需要进一步解决的缺点                     |
| 关键问题 | 关键缺点转化后的问题表达，例如“道路交会过多”转为“如何减少道路交会” |

因果链步骤：

1. 列出目标问题或初始缺点。
2. 对每个缺点逐级列出直接原因。
3. 用 `And/Or` 表达同层缺点关系。
4. 重复查找直到末端缺点。
5. 检查功能分析、成本分析、流分析得到的缺点是否已包含。
6. 确定关键缺点。
7. 将关键缺点转为关键问题，寻找方案。
8. 从关键问题出发挖掘可能存在的矛盾。

注意：中间缺点要找直接原因，不要跳跃。例如“手被水杯烫伤”不能直接跳到“水温高”，应分析为“手表面温度高-手与杯接触且杯面温度高-杯导热性高且水温高”。

### 3.3 鱼骨图分析法

- 用于查找原因。
- 常从 `人、机、料、法、环、测` 六个维度展开。
- 构建步骤：
    1. 把要解决的问题、特性或结果放在鱼头。
    2. 画出主要原因分支。
    3. 逐个方面讨论原因。
    4. 对子分支继续深入。
    5. 从重要性和困难程度评估方案。
- 三种类型：
    - 原因型：鱼头常写“为什么……”
    - 对策型：鱼头常写“如何提高/改善……”
    - 整理问题型：用于结构化整理，不一定是因果关系。

## 4. 功能分析

### 4.1 基本概念

| 概念          | 含义                                                         |
| ------------- | ------------------------------------------------------------ |
| 技术系统      | 由相互联系的物质及相互作用组成，用于实现某种功能             |
| 超系统        | 把当前技术系统作为组件的更大系统，或与技术系统相关但不属于其本身的系统 |
| 组件          | 技术系统或超系统中的组成部分，可以由物质或场组成             |
| 目标/作用对象 | 技术系统的作用对象                                           |

功能定义：某组件改变或保持另一组件某个参数的行为或作用。

标准表达：

`X 更改或保持 Z 的参数 Y`

其中：

- `X` 是功能载体。
- `Z` 是功能对象。
- `Y` 是功能对象的某个参数。

### 4.2 功能描述原则

重点背：

1. 用动宾词组描述：`功能 = 工作/动作 + 对象`，即 `V + O`。
2. 功能受体至少有一个参数被影响或改变。
3. 禁止用“不”替代否定动词，例如不要说“陶瓷不能传导电流”，应说“陶瓷阻碍电流”。
4. 功能对象必须是组件，不能是组件参数。
5. 针对特定技术系统和具体条件陈述。
6. 尽量使用抽象表达，避免过度专业术语。

直觉表达与本质表达要区分：

| 技术系统 | 直觉表达   | 本质表达       |
| -------- | ---------- | -------------- |
| 电吹风   | 吹干头发   | 蒸发水分       |
| 电风扇   | 凉爽身体   | 移动空气       |
| 放大镜   | 放大目标物 | 改变光线       |
| 白炽灯   | 照亮房间   | 发光           |
| 二极管   | 整流电流   | 阻滞某极性电流 |

### 4.3 功能分类

| 分类维度 | 类型                                | 说明                                                         |
| -------- | ----------------------------------- | ------------------------------------------------------------ |
| 作用结果 | 有用功能                            | 按期望方向改善对象参数                                       |
| 作用结果 | 有害功能 H                          | 不按期望方向改变对象参数，甚至恶化                           |
| 功能等级 | 基本功能 B                          | 对象是系统目标，是系统存在主要理由，价值最高，常记 3 分      |
| 功能等级 | 辅助功能 Ax                         | 对象是系统组件，用于支撑基本功能，价值常记 1 分；若对象是提供基本功能的组件，可记 2 分 |
| 功能等级 | 附加功能 Ad                         | 对象是超系统组件，回答“系统还能做什么”，价值常记 2 分        |
| 性能水平 | 正常 N                              | 实际改善值达到期望值                                         |
| 性能水平 | 过度 E                              | 实际改善值大于期望值                                         |
| 性能水平 | 不足 I                              | 实际改善值小于期望值                                         |
| 成本水平 | 微不足道 Ne、可接受 Ac、难以接受 UA | 用于评估功能成本                                             |

### 4.4 功能整理

功能整理：对定义出的功能进行系统分析和整理，明确功能关系、类别，建立功能模型图。

步骤：

1. 分析产品的基本功能和辅助功能。
2. 明确功能上下位关系和并列关系。
3. 排列功能模型图。

常用提问：

- 为什么需要此功能？用于找上位功能，也就是目的。
- 通过什么手段实现此功能？用于找下位功能，也就是手段。

FAST 图解法：

- 适用于开发新技术系统。
- 以“目的-手段”逻辑逐层展开。
- 从右向左问“为什么”，从左向右问“怎么办”。
- 越靠近目的界限的功能越重要。

### 4.5 组件式系统功能分析

适用于改进已有系统。

主要分析内容：

1. 确定技术系统主功能。
2. 研究各组件对系统功能的贡献。
3. 分析有用功能和有害功能。
4. 对有用功能确定等级和性能水平。
5. 建立组件功能模型和功能模型图。

三步流程：

1. 组件拆分：识别系统组件、子系统组件、作用对象和超系统组件。
2. 相互作用分析：用相互作用矩阵判断两两组件是否存在作用。
3. 功能模型建立：列出功能、功能等级、性能水平，绘制功能模型。

组件拆分注意事项：

- 选择合理分析层级。
- 尽量在同一层级选择组件，不要混杂。
- 多个相同组件可视为一个组件。
- 需要更细分析时，可把某组件放到低一层级重新拆分。
- 超系统组件不是技术系统一部分，但与其相互影响。
- 组件数量尽量控制在 10 个以内，否则分析过于复杂。

相互作用分析：

- 接触作用和场作用都算相互作用。
- 矩阵中有作用记 `+`，无作用记 `-`。
- 若某组件与其他任何组件都没有相互作用，复核后可直接裁剪。

功能模型图形：

- 系统组件：矩形框。
- 超系统组件：六边形。
- 系统作用对象：圆角矩形。
- 箭头：表示功能作用，箭头文字用动词或动宾短语。

### 4.6 系统功能分析的作用

- 明确发明对象应具有的全部功能。
- 为创造方案提供依据。
- 掌握功能之间的逻辑关系和相互影响。
- 扩大方案创造思路：从结构思维转为功能思维。

## 5. 系统裁剪

系统裁剪：当系统需要删减某些组件时，把其有用功能提取出来，让系统中已有或周边其他部分完成该功能，从而降低成本、提高理想度。

### 5.1 裁剪目的

- 精减组件数量，降低组件成本。
- 优化功能结构，合理布局系统架构。
- 体现功能价值，提高功能实现效率。
- 消除过度、有害、重复功能，提高系统理想化程度。

### 5.2 裁剪前思考

1. 是否需要该组件提供的功能？
2. 系统内部或周边是否有其他组件可以实现该功能？
3. 现有资源能否实现该功能？
4. 能否用更便宜的方法实现该功能？
5. 该组件与其他组件之间是否存在必要装配或运动关系？

### 5.3 裁剪对象选择原则

按项目目标选择：

- 降低成本：优先选择功能价值低、成本高的组件。
- 专利规避：优先选择专利权利要求相关组件。
- 改善系统：优先选择有主要缺点的组件。
- 降低复杂度：优先选择复杂度高的组件。

其他优先对象：

- 具有有害功能的组件。
- 低价值组件。
- 提供辅助功能的组件。

### 5.4 裁剪三规则

确定欲裁剪组件 `A` 后，按顺序尝试：

1. 若移除作用对象 `B`，则功能载体 `A` 可被裁剪。
2. 功能对象 `B` 能自我完成 `A` 的有用功能。
3. 技术系统或超系统中的其他组件 `C` 能完成 `A` 的功能，`C` 可以是已有组件，也可以是新增组件。

优先级：`规则 1 -> 规则 2 -> 规则 3`。

经典例子：近视眼镜裁剪

- 裁剪镜腿：让鼻子或手支撑镜框，形成无腿眼镜。
- 裁剪镜框：让眼睛支撑镜片，形成隐形眼镜思路。
- 裁剪镜片：让眼睛本身改变光线方向，形成近视眼手术思路。

## 6. 资源分析

TRIZ 观点：创新过程就是资源的运用与重新配置过程。应用系统或超系统资源解决问题，是 TRIZ 的基本观点。

### 6.1 发明资源定义

发明资源包括：

- 系统或环境中已有物质，包括废物，经变换后得到的新物质。
- 储存能量、自由时间、未占用空间、信息等。
- 实现附加功能的技术能力，包括物质特性、物理/几何/其他效应。

### 6.2 资源分类

按可及性和所在区域：

- 内部资源：冲突区域内部资源。
- 外部资源：冲突区域外部但系统内资源。
- 超系统资源。

按可用状态：

- 直接应用资源。
- 导出资源/衍生资源。
- 差动资源。

按资源形式：

| 类型     | 例子                                                         |
| -------- | ------------------------------------------------------------ |
| 物质资源 | 组成元素、原材料、产物、废弃物、环境中廉价物质               |
| 场资源   | 系统内能源、系统周围能源、可转化利用资源                     |
| 信息资源 | 物质或场的属性、内部信息流、状态改变信息、差异性信息、临时性信息 |
| 功能资源 | 系统主要功能、有害作用的利用、元件或系统潜在功能             |
| 时间资源 | 作用前时间、作用后时间、间歇时间、压缩产生的时间、并行时间   |
| 空间资源 | 闲置空间、空洞、由几何变化或效应产生的额外空间               |

### 6.3 资源搜索方向

可按以下方向搜索：

- 搜索辅助功能元件资源。
- 搜索与工具相关资源。
- 搜索与制品相关资源。
- 搜索有害资源、废弃物。
- 搜索直接应用资源。
- 搜索导出资源。
- 搜索差动资源。
- 应用资源列表。
- 应用资源组合。
- 在超系统搜索资源。

### 6.4 资源选择原则

1. 可及性：内部资源优先，其次系统资源，再次超系统或环境资源。
2. 可用性及成本：优先选择成本低、容易使用的资源。
3. 数量：优先过剩资源，其次供应充足资源，最后供应不足资源。
4. 质量：优先有用资源，其次不确定资源，最后有害资源。

## 7. 最终理想解 IFR

### 7.1 理想化

TRIZ 解题时先抛开客观限制，通过理想化定义最终理想解 `IFR`，明确问题解决方向，避免无目标试错。

常见理想化表述：

- 理想系统：没有实体、不消耗能源，却能实现所有需要功能。
- 理想过程：只有过程结果，没有过程本身。
- 理想资源：资源无穷无尽，随意使用且不付费。
- 理想方法：不消耗能量和时间，通过自身调节获得所需功能。
- 理想机器：没有质量和体积，却完成所需工作。
- 理想物质：没有物质，但功能实现。

### 7.2 理想度公式

基础公式：

`理想度 = 有用功能之和 / 有害作用之和`

工程公式：

`理想度 = 有用功能之和 / (有害作用之和 + 成本之和)`

### 7.3 提高理想化水平的方向

- 增加新的功能，或从超系统获得功能。
- 把尽可能多的功能传输到工作元件上。
- 利用内部或外部已有资源，尤其是超系统免费资源。
- 剔除无效或低效率功能。
- 预防有害功能，将有害功能转化为中性功能，减轻有害等级。
- 将有害功能移到超系统中，使其不再成为本系统的有害功能。

### 7.4 理想化方法

部分理想化：在选定原理上，通过不同实现方式使系统理想化。常用 6 种模式：

1. 加强有用功能。
2. 降低有害功能。
3. 功能通用化。
4. 个别功能专用化。
5. 增加集成度。
6. 增加柔性。

全部理想化：同一功能选择不同原理，使系统理想化。通常在部分理想化无效后使用。4 种模式：

1. 功能的剪切。
2. 系统的剪切。
3. 原理的改变。
4. 系统换代。

### 7.5 IFR 的特点和确定步骤

最终理想解 4 个特点：

1. 保持原系统优点。
2. 消除原系统不足。
3. 不使系统变复杂。
4. 不引入新的缺陷。

确定 IFR 的 5 个问题：

1. 设计的最终目的是什么？
2. 理想解是什么？
3. 达到理想解的障碍是什么？
4. 出现这种障碍的结果是什么？
5. 不出现这种障碍的条件是什么？创造这些条件的可用资源是什么？

## 8. 技术矛盾与 40 个发明原理

### 8.1 矛盾分类

| 类型     | 含义                                                         | 是否经典 TRIZ 重点     |
| -------- | ------------------------------------------------------------ | ---------------------- |
| 管理矛盾 | 想取得结果或避免现象，但不知道怎么做                         | 不是经典 TRIZ 研究重点 |
| 技术矛盾 | 一个作用同时产生有用和有害结果；或消除有害作用导致其他功能变坏 | 是                     |
| 物理矛盾 | 为实现功能，同一子系统/元件需要一种特性，同时又需要相反特性  | 是                     |

技术矛盾出现的三种情况：

1. 引入有用功能，导致另一子系统产生或加强有害功能。
2. 消除有害功能，导致另一子系统有用功能变坏。
3. 有用功能加强或有害功能减少，使另一子系统或系统变复杂。

### 8.2 技术矛盾标准化表达

标准句式：

`如果采用某措施，那么实现某改进目标，但是导致某不期望结果。`

例：

`如果增大发动机整流罩直径，那么发动机进气量增大，但是发动机离地距离减小，安全性降低。`

其中：

- 改进目标对应改进参数。
- 不期望结果对应恶化参数。
- 二者构成技术矛盾。

### 8.3 技术矛盾分析过程

1. 明确改进目的，从 39 个通用工程参数中选改进参数。
2. 根据经验、常用方法或效应知识库确定达到目的的途径。
3. 分析采用该途径后产生的不期望结果，选恶化参数。
4. 改进参数和恶化参数形成一对或多对冲突。
5. 查冲突矩阵，获得推荐发明原理。
6. 根据发明原理形成具体工程方案。

## 9. 物理矛盾与分离原理

### 9.1 物理矛盾定义

物理矛盾：对系统同一个元素或同一个参数提出不同甚至相反的要求，常用 `A+ / A-` 表示。

例子：

- 筛子/口罩/纱窗：孔径要小，过滤效果好；孔径要大，不易堵塞。
- 飞机起落架：起飞降落时需要；飞行过程中不需要。
- 钢笔笔尖：要细，写细字；又要粗，避免划破纸。
- 手机天线：为了信号好要有天线；为了体积小又不要天线。

### 9.2 物理矛盾三种情况

1. 同一工程参数在不同设计条件下有相反要求。
2. 同一工程参数在不同工况下有不同要求，不一定相反。
3. 同一非工程参数在不同工况下有不同要求。

### 9.3 技术矛盾与物理矛盾的关系

| 比较 | 技术矛盾              | 物理矛盾                     |
| ---- | --------------------- | ---------------------------- |
| 对象 | 两个参数之间的矛盾    | 同一元素/同一参数的不同要求  |
| 层次 | 更显而易见            | 更深层、更尖锐，常是本质矛盾 |
| 工具 | 矛盾矩阵、40 发明原理 | 分离方法、对应发明原理       |

很多技术矛盾可以进一步转化为物理矛盾。

例：安瓿瓶密封

- 技术矛盾：高温火焰提高生产率，但增加有害因素。
- 物理矛盾：温度既要高以提高生产率，又要低以减少有害因素。

### 9.4 四类分离原理

| 分离方法     | 核心思想                             | 判断问题                                             |
| ------------ | ------------------------------------ | ---------------------------------------------------- |
| 空间分离     | 同一参数的不同要求在不同空间实现     | 哪个空间需要 A+？哪个空间需要 A-？两个空间是否交叉？ |
| 时间分离     | 同一参数的不同要求在不同时间段实现   | 什么时间需要 A+？什么时间需要 A-？两个时间是否交叉？ |
| 条件分离     | 同一参数的不同要求在不同条件下实现   | 是否可以按速度、温度、载荷、环境等条件切换？         |
| 系统级别分离 | 同一参数的不同要求在不同系统层级实现 | 能否在子系统/系统/超系统不同层级满足相反要求？       |

### 9.5 分离方法与创新原理对应

| 分离方法     | 常用创新原理                                                 |
| ------------ | ------------------------------------------------------------ |
| 空间分离     | 1 分割、2 抽取、3 局部质量、4 增加不对称性、7 嵌套、13 逆向思维、17 多维化、24 中介物、26 复制、30 柔性外壳或薄膜 |
| 时间分离     | 9 预先反作用、10 预先作用、11 事先防范、15 动态特性、16 不足或过度作用、18 机械振动、19 周期性动作、20 有效作用连续性、21 减少有害作用时间、29 气压与液压结构、34 抛弃与再生、37 热膨胀 |
| 条件分离     | 40 复合材料、31 多孔材料、32 改变颜色、3 局部质量、19 周期性作用、17 一维变多维 |
| 系统级别分离 | 1 分割、5 组合、33 同质性、12 等势性                         |

### 9.6 解题模板

1. 描述问题并定义理想解。
2. 定义物理矛盾：
    - 参数是什么？
    - 要求 1 是什么？
    - 要求 2 是什么？
3. 判断该参数的不同要求能否在空间、时间、条件或系统级别上分离。
4. 选择对应分离方法。
5. 从对应创新原理中寻找启发。
6. 形成具体方案。

## 10. 物场分析

### 10.1 物场模型概念

物场分析 `Substance-Field Analysis` 是 TRIZ 中针对问题建模分析的工具。

技术系统存在的目的在于实现功能，而功能可以分解为：

- `S1`：物质 1，通常表示工件、作用对象、被动作用体。
- `S2`：物质 2，通常表示工具、主动作用体。
- `F`：场，表示 S1 和 S2 之间的相互作用或效应。

完整功能可表示为一个 `S1-S2-F` 三角形。

例：

- 洗衣机洗衣服：S1=衣服，S2=洗衣机，F=机械场/清洗作用。
- 手拿杯子：S1=杯子，S2=手，F=机械场。

### 10.2 场的类型

常见场：

- 重力场 G
- 机械场 Me
- 气动场 P
- 液压场 H
- 声场 A
- 热场 Th
- 化学场 Ch
- 电场 E
- 磁场 M
- 光学场 O
- 放射场 R
- 生物场 B
- 粒子场 N

### 10.3 物场模型四种类型

| 类型         | 特点                         | 处理目标             |
| ------------ | ---------------------------- | -------------------- |
| 有效完整模型 | 三元素都存在且作用有效充分   | 理想状态，通常不需改 |
| 不完整模型   | 缺少场或物质                 | 补齐元素             |
| 效应不足模型 | 三元素存在，但作用不足或无效 | 强化或替换作用       |
| 有害效应模型 | 三元素存在，但产生有害作用   | 消除或抵消有害作用   |

TRIZ 关注后三类问题模型，目标是把它们转化为有效完整模型。

### 10.4 六种一般解法

| 解法       | 内容                                             | 对应模型     |
| ---------- | ------------------------------------------------ | ------------ |
| 一般解法 1 | 补齐元素，增加场或物质                           | 不完整模型   |
| 一般解法 2 | 增加第三种物质 S3 阻止有害作用                   | 有害效应模型 |
| 一般解法 3 | 引入另一个场 F2 抵消原场有害效应                 | 有害效应模型 |
| 一般解法 4 | 用另一个场 F2 替代原场 F                         | 效应不足模型 |
| 一般解法 5 | 增加另一个场 F2 强化有用效应                     | 效应不足模型 |
| 一般解法 6 | 引入第三种物质 S3 并增加另一个场 F2 强化有用效应 | 效应不足模型 |

### 10.5 物场分析步骤

1. 识别元件：确定 S1、S2、F。
2. 构造模型：判断是完整、不完整、效应不足还是有害效应。
3. 选择方法：根据模型类型选择一般解法或标准解。
4. 发展概念：结合实际限制形成具体解决方案。

## 11. 标准解

### 11.1 标准解定义

标准解系统主要用于条件和约束确定后的发明问题解决，针对物场模型进行分析。

关系：

- 物场分析发现问题并指出方向。
- 标准解在该方向下提供更具体、更系统的解决方案。
- 阿奇舒勒提出 5 级共 76 个标准解。

编号规则：

`S N.M.X`

- `S`：标准解。
- `N`：级。
- `M`：子级。
- `X`：具体解。

### 11.2 标准解五级

| 级别    |  数量 | 核心用途                                                     |
| ------- | ----: | ------------------------------------------------------------ |
| 第 1 级 | 13 个 | 建立和拆解物场模型；不改变或少量改变系统，改进输出或消除不理想输出 |
| 第 2 级 | 23 个 | 完善物场模型；通过较大改变改善系统                           |
| 第 3 级 |  6 个 | 系统传递；向双系统、多系统、超系统或微观级别转化             |
| 第 4 级 | 17 个 | 检测与测量问题                                               |
| 第 5 级 | 17 个 | 标准解优化；简化或改进前述标准解                             |

### 11.3 标准解与一般解法关系

- 一般解法更抽象，主要指出方向。
- 标准解更具体，给出更系统的解决路径。
- 对应关系大致为：
    - 不完整模型：第 1.1 级标准解。
    - 有害完整模型：第 1.2 级标准解。
    - 效应不足模型：第 2 级等标准解。
    - 检测测量问题：第 4 级标准解。
    - 模型优化：第 5 级标准解。

### 11.4 各级重点

第 1 级：改进系统

- 完善不完整物场。
- 在 S1/S2 内部或外部引入添加物。
- 利用环境资源。
- 拆解物场模型，消除或抵消有害效应。

第 2 级：改变系统

- 传递到复杂物场模型：串联、并联。
- 加强物场模型：使用易控制的场、宏观转微观、多孔材料、提高柔性。
- 频率匹配或不匹配。
- 铁磁材料与磁场结合。

第 3 级：系统传递

- 单系统转双系统或多系统。
- 提高双/多系统动态性。
- 增加系统元件差异。
- 简化双/多系统。
- 利用整体与部分相反特性。
- 转换到微观水平。

第 4 级：检测与测量

- 修改系统，使其不再需要检测与测量。
- 用复制品或图像替代对象。
- 用两次测量代替连续测量。
- 创造或合成测量系统。
- 利用自然现象、谐振、铁磁场等加强测量。

第 5 级：优化

- 引入物质：优先无成本资源，如空气、真空、泡沫、空洞。
- 用场代替物质。
- 引入临时添加物，用后自动消除。
- 使用环境中已有场。
- 应用相变、自然现象和物质粒子分解/组合。

## 12. 技术系统进化法则

### 12.1 系统与技术系统

系统：若干相互联系、相互作用的部分组成的，在一定环境中具有特定功能的有机整体。

系统特性：

- 整体性
- 相关性
- 层次性
- 环境制约性与适应性
- 有序性
- 动态性、复杂性、随机性
- 目的性

技术系统：能够提供人们所需功能的相互联系要素组合。

技术系统三特征：

1. 人造系统。
2. 能为人类提供某种功能。
3. 由多种元素组成。

### 12.2 八大进化法则

1. 提高理想度法则：增加有用功能，减少有害作用和成本。
2. 系统完备性法则：完整技术系统包含动力、传输、执行、控制四个子系统。
3. 能量传递法则：能量应从能量源流向所有元件，进化方向是缩短能量路径、减少转换和损失。
4. 子系统不均衡进化法则：各子系统沿各自 S 曲线发展，系统速度取决于最不理想的短板。
5. 增加动态性和可控性法则：由不可动到可动，由刚性到柔性，由手动到自动/自适应。
6. 向超系统进化法则：单系统转双系统、多系统，或把某子系统功能转移到超系统。
7. 子系统协调性进化法则：形状、性能参数、节奏或频率协调。
8. 向微观级和场的应用进化法则：从宏观结构向微观结构、从机械作用向高效场作用进化。

### 12.3 S 曲线

S 曲线描述技术系统生命周期和重要性能参数变化规律。

四阶段：

| 阶段   | 特征                                           |
| ------ | ---------------------------------------------- |
| 婴儿期 | 效率低，可靠性差，投入不足，发展缓慢           |
| 成长期 | 价值和潜力显现，投入增加，效率和性能快速提高   |
| 成熟期 | 系统完善，性能达到较高水平，利润最大但可能下降 |
| 衰退期 | 技术接近极限，难以突破，将被新技术系统替代     |

应用：

- 产生市场需求。
- 定性技术预测。
- 产生新技术。
- 实施专利布局。
- 选择企业战略时机。

## 13. 科学效应知识库

科学效应库用于把“需要实现的功能”映射到可用科学效应。

应用步骤：

1. 根据问题实际情况，定义需要的功能。
2. 根据功能从 How-to 表中确定功能代码，例如 `F1-F30`。
3. 从效应表中查找该功能代码，得到推荐科学效应。
4. 对推荐效应逐一筛选，找到适合本问题的效应。
5. 查找效应详细解释，应用于问题解决，形成方案。

课件示例：

- 麦比乌斯圈：研磨带工作面增加，寿命延长。
- 超塑性：适当温度和压力下形成复杂中空元件。
- 离子束溅射：用于电子发射器制造。
- 液体中物体重量减少：利用浮力降低加速度影响。

## 14. 高频易混点

### 14.1 技术矛盾 vs 物理矛盾

- 技术矛盾：两个参数之间，一个改善、一个恶化。
- 物理矛盾：同一个参数既要这样，又要相反或不同。
- 技术矛盾通常先用 39 参数和矛盾矩阵；物理矛盾通常用四类分离。

### 14.2 功能模型 vs 物场模型

- 功能模型：围绕组件之间的功能作用，强调“谁对谁做什么”，用于系统功能分析和裁剪。
- 物场模型：把功能抽象为 `S1-S2-F`，强调物质和场，常用于标准解。

### 14.3 一般解法 vs 标准解

- 物场六种一般解法：方向性强，适合快速判断。
- 76 个标准解：更系统、更细化，适合在物场模型已经明确后检索方案。

### 14.4 IFR vs 理想度

- IFR：针对具体问题的最终理想状态，是解题目标。
- 理想度：衡量系统好坏的指标，通常为有用功能与有害作用/成本的比值。

### 14.5 功能分析 vs 因果分析

- 功能分析：找系统组件、功能、有害/不足/过度功能。
- 因果分析：沿原因链找根因和关键缺点。
- 实际解题常先功能分析定位问题功能，再因果链深挖根因。

## 15. 案例题答题模板

### 15.1 功能分析题

1. 确定技术系统、系统目标、系统组件、超系统组件。
2. 组件拆分，控制在合理层级和数量。
3. 建立相互作用矩阵。
4. 用 `动词 + 对象` 描述功能。
5. 标注功能等级：B、Ax、Ad、H。
6. 标注性能水平：N、E、I。
7. 绘制或文字描述功能模型。
8. 找出有害、过度、不足、低价值功能，提出裁剪或改进方向。

### 15.2 技术矛盾题

1. 写出原始问题。
2. 用句式表达：如果采用某措施，那么改善什么，但是恶化什么。
3. 把改善目标映射到 39 参数。
4. 把恶化结果映射到 39 参数。
5. 查矛盾矩阵得到发明原理。
6. 结合原理形成具体方案。

### 15.3 物理矛盾题

1. 找同一元素/参数。
2. 写出相反或不同要求：`A+` 和 `A-`。
3. 判断可否按空间、时间、条件、系统级别分离。
4. 选择对应发明原理。
5. 形成方案，并说明为什么满足两种要求。

### 15.4 物场分析题

1. 确定 S1、S2、F。
2. 判断模型类型：不完整、有效不足、有害效应或有效完整。
3. 根据类型选六种一般解法。
4. 必要时查标准解。
5. 把抽象解法转成实际工程方案。

### 15.5 最终理想解题

1. 最终目的是什么？
2. 最理想结果是什么？
3. 障碍是什么？
4. 障碍造成什么后果？
5. 不出现障碍需要什么条件？
6. 可用资源是什么？
7. 检查是否保持优点、消除不足、不增加复杂度、不引入新缺陷。

## 17. 考前背诵压缩版

- TRIZ：通过专利和创新案例总结出来的系统化发明问题解决理论。
- TRIZ 三核心：技术系统有进化规律；矛盾解决推动进化；用尽量少资源实现尽量多功能。
- 问题：期望状态与当前状态的差距。
- 发明问题：至少一个关键步骤未知、目标不清或存在矛盾需求。
- 5WHY：从结果沿因果链追问到根因。
- 因果链：初始缺点 -> 中间缺点 -> 末端缺点 -> 关键缺点 -> 关键问题。
- 功能：组件改变或保持另一组件某参数的作用，表达为 `动词 + 对象`。
- 功能等级：基本 B、辅助 Ax、附加 Ad、有害 H。
- 性能水平：正常 N、过度 E、不足 I。
- 功能分析：组件拆分 -> 相互作用分析 -> 功能模型建立。
- 裁剪：删组件、保功能、提高理想度。
- 资源：物质、场、信息、功能、时间、空间；内部优先、低成本优先、过剩优先。
- IFR：保持优点、消除不足、不复杂、不引入新缺陷。
- 技术矛盾：一个参数改善，另一个参数恶化。
- 物理矛盾：同一参数有相反或不同要求。
- 技术矛盾工具：39 参数 + 矛盾矩阵 + 40 发明原理。
- 物理矛盾工具：空间、时间、条件、系统级别分离。
- 物场模型：S1、S2、F。
- 物场问题：不完整、效应不足、有害效应。
- 标准解：5 级 76 个，比一般解法更具体。
- 进化法则：理想度、完备性、能量传递、不均衡、动态可控、超系统、协调性、微观和场。
