# C - Count Close Pairs

> 关键词：交互，双指针

## 思路

设第 $i$ 个点的位置为 $x_i$，题目保证 $x_1 < x_2 < \cdots < x_N$，询问 $(i,j)$ 实际上是在判断 $x_j-x_i \le 1$，对于固定的左端点 $i$，随着 $j$ 从小到大移动，距离 $x_j-x_i$ 单调增加。因此一定存在一个分界点，使得 $(i,i+1),(i,i+2),\ldots,(i,r_i)$ 的距离都不超过 $1$，而分界点后的距离都大于 $1$。所以，以 $i$ 为左端点的合法点对数量就是 $r_i-i$。

考虑如何找到所有的 $r_i$，假设对于点 $i$，点 $j$ 与它的距离不超过 $1$，即 $x_j-x_i\le1$，因为 $x_{i+1}>x_i$，所以 $x_j-x_{i+1}<x_j-x_i\le 1$。也就是说，如果点 $j$ 与点 $i$ 足够近，那么点 $j$ 与点 $i+1$ 也一定足够近。因此 $r_1\le r_2\le r_3\le\cdots\le r_N$，右边界 $r_i$ 单调不降。

使用双指针，维护当前左端点 $l$ 和对于当前左端点 $l$，最靠右的右端点 $r$，对于每个 $l$，保证 $r \ge i + 1$，询问 $(i,j)$。如果距离不超过 $1$，说明这个点对合法，让 $j$ 继续右移。如果距离大于 $1$，说明 $j$ 是第一个不合法的位置，对于当前的 $l$，答案为 $r-l$。总答案为 $\sum_{i=1}^{N}(r_i-i)$。

> [!NOTE]
>
> 注：交互题的输出使用 `<< flush` 或者 `<< endl` 刷新缓存。

```c++
using pdd = pair<double, double>;
using pll = pair<long long, long long>;
```

# D - Placing Rooks

> 关键词：模拟，倒序

## 思路

正着模拟很麻烦，考虑一个棋子什么时候可以活到最后被计入答案。

我们发现对于第 $i$ 次操作，最终放在 $(R_i,C_i)$ 的棋子，要一直保留到最后，需要满足：

- 后面的操作没有再次清空第 $R_i$ 行；
- 后面的操作没有再次清空第 $C_i$ 列。

因此倒序操作，并维护每行每列是否在后面的操作中出现过。当遍历到某次操作的行列均未被后面访问过，说明后面既没有清空这一行，也没有清空这一列，所以本次放下的棋子能够保留到最后。

**注意，每次操作的行和列都必须标记，即使当前操作放下的棋子最终不能保留，这次操作依然清空过对应的行和列，所以它仍然会影响更早放下的棋子。**

# E - Range Flip

> 关键词：DP

## 思路

[动态规划 (Notebook) - 经典题 - 选择m个不相交区间使区间和最大](https://blog.hailuo4ever.com/posts/notebook/dp/#选择m个不相交区间使区间和最大)

初始所有卡牌正面朝上，基础答案为 $\text{base}=\sum_{i=1}^{N}A_i$。如果第 $i$ 张牌最终翻到背面，它的数值由 $A_i$ 变成 $B_i$，答案的变化量 $C_i=B_i-A_i$​。

因此，假设最终翻到背面的牌构成若干区间，那么这些区间带来的额外收益就是对应 $C_i$ 的区间和。由于最多操作 $K$ 次，最终背面朝上的位置可以表示成至多 $K$ 个互不相交区间。

答案为 $\sum_{i=1}^{N}A_i+g[N][K]$。

## Code

```c++
// Problem: AT ABC466 E
// Contest: AtCoder - AtCoder Beginner Contest 466
// URL: https://atcoder.jp/contests/abc466/tasks/abc466_e
// Time: 2026-07-11 20:00:18
#include <bits/stdc++.h>
using namespace std;

// clang-format off
#define endl '\n'
#define all(x) (x).begin(), (x).end()
#define fastio() ios::sync_with_stdio(0); cin.tie(0); cout.tie(0);
// clang-format on

using ll = long long;
using ull = unsigned long long;
using pii = pair<int, int>;
using pdd = pair<double, double>;
using pll = pair<long long, long long>;
using i128 = __int128;

const int dx[] = {-1, 0, 1, 0, -1, 1, 1, -1};
const int dy[] = {0, 1, 0, -1, 1, 1, -1, -1};
const int inf = 0x3f3f3f3f;
const ll INF = 1e18;
const int N = 0;

void solve()
{
    int n, K;
    cin >> n >> K;

    vector<ll> a(n + 1), b(n + 1), gain(n + 1);

    for (int i = 1; i <= n; i++)
        cin >> a[i] >> b[i];

    ll base = 0;
    for (int i = 1; i <= n; i++)
    {
        base += a[i];
        gain[i] = b[i] - a[i];
    }

    vector<vector<ll>> f(n + 1, vector<ll>(K + 1, -INF));
    vector<vector<ll>> g(n + 1, vector<ll>(K + 1, 0));

    for (int i = 1; i <= n; i++)
    {
        for (int j = 1; j <= K; j++)
        {
            f[i][j] = max(f[i - 1][j], g[i - 1][j - 1]) + gain[i];
            g[i][j] = max(g[i - 1][j], f[i][j]);
        }
    }

    cout << base + g[n][K] << endl;
}

int main()
{
    fastio();

    int T = 1;
    // cin >> T;

    while (T--)
        solve();

    return 0;
}
```
